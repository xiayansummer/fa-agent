从下面的会议纪要里抽取所有 action items / milestone（含 IR 侧与投资人侧），输出严格 JSON 数组。

会议纪要：
{full_minutes}

## 当前参考时间
{today_iso}

## 输出格式
严格 JSON 数组，不要 markdown 代码块包裹。每项格式：
{{
  "title": "10-30 字简短描述。投资人侧请带上人名，例如 '杨阳出 TS' / '张慧明完成立项'",
  "actor": "ir" | "investor",
  "type": "send_material" | "followup_call" | "meeting_request" | "investor_milestone" | "other",
  "due_date": "ISO 8601 日期 YYYY-MM-DD"
}}

## 规则
1. 抽两类事件：
   - **IR 要做的事** (actor="ir")：发资料 / 跟进电话 / 约见面 等
   - **投资人要做的事** (actor="investor")：出 TS / 立项 / 给反馈 / 完成 DD / 决策 / 内部评审 等
   关键判别：动作主语是"我们/IR/我"→ ir；主语是"他们/投资人/机构/某某总" → investor。

2. due_date 必填，推算：
   - "今天"/"明天" → 对应日期
   - "两天内"/"三天后" → 今天 + 天数
   - "下周" → 今天 + 5~7 天
   - "下下周"/"两周后" → 今天 + 12~14 天
   - 没明确时间 → 今天 + 3 天（IR 侧）/ 今天 + 7 天（投资人侧，给对方更宽窗口）

3. type 推断：
   - IR 发资料/DataPack/Term Sheet/PPT → send_material
   - IR 电话/微信联系 → followup_call
   - IR 约见面/路演/线下 → meeting_request
   - 投资人侧任意事项 → investor_milestone
   - 其它 → other

4. actor 一定要和 type 一致：
   - send_material / followup_call / meeting_request → actor 必为 ir
   - investor_milestone → actor 必为 investor

5. 没有 action 时输出 []。最多 5 条。

只输出 JSON，不要任何解释。
